为什么0.1 + 0.2 不等于 0.3 ?

计算机基础
计算机基础
发布日期:   2020-08-27
文章字数:   1.8k

IEEE浮点标准

要解答这个问题，我们需要一些前置基础知识。现在的浮点型的存储基本按IEEE754标准实现，浮点数分为单精度32位和双精度64位。

IEEE浮点标准用 V = ( -1) s * M * 2E 的形式表示一个数：

  • 符号 s 决定这数是负数**(s = 1)还是正数(s = 0)**，而对于数值0的符号位解释作为特殊情况处理
  • 尾数 M是一个二进制小数，它的范围是 1 ~ 2 - ε 或者 0 ~ 1 - ε
  • 阶码 E的作用是对浮点数加权，这个权重是2的E次幂(可能是负数)

从 V = ( -1) s * M * 2E 我们可以知道，要在计算机表示一个浮点数需要 S, M, E 三个部分，在计算机底层位的表示我们要划分出三个字段分别表示它们的值。下图分别是单精度和双精度的标准浮点格式。

下面我以单精度为例，解释各部分的意思：

  • s 占1位，用0表示正数，用1表示负数 直接编码 V = ( -1) s * M * 2E 中的 s
  • 用k位(30 - 23 + 1 = 8位)**的阶码字段exp = ek -1 … e1e0** 来编码 V = ( -1) s * M * 2E 中的阶码E
  • 用n(22 - 0 + 1 = 23)位小数字段 frac = fn -1….f1f0 来编码尾数M，但是编码出来的值也依赖于阶码字段的值是否等于0

给定位表示，根据exp的值，被编码的的值可以分为三种不同的情况(最后一种情况有两种变种)。下图说明了对单精度格式的情况。

  1. 规格化的值

    当exp的位不全为0，也不全为1，都属于这种情况。在这种情况中，阶码字段被解释为以偏置(biased)形式表示的有符号整数。什么叫偏置形式表示的有符号整数？其实简单来说就是： 我们阶码的值是E= e - Bias，首先，Bias(偏置值) = 2(n - 1) -1 = 127，e是无符号数，所以e能表示的最小整数为1，最大整数为255，所以能表示指数的范围为**(1 - 127) ~ (254 - 127) 即 (-126 ~ 127)**
    小数字段frac被解释为描述小数值f(0 <= f < 1) ，用二进制表示f = 0.f(n - 1)…f1f0 尾数的定义为M = 1 + f 。可以观察看出M = 1.f(n - 1)…f1f0 ，由于我们只需在0~(n -1)位中填入 f(n - 1)…f1f0 ，所以这里隐含以1开头的表示(第一位永远是1)，也就是说我们用 f(n - 1)…f1f0 来表示 1.f(n - 1)…f1f0 不需要显式表示它，我们可以轻松获得一个额外精度。

  2. 非规格化的值

    当阶码全为0时，表示非格式化形式。这种情况，阶码值是E= 1 - Bias，尾数的值是M = f，也就是小数字段的值，不包含隐含开头的1
    非规格化数有两个用途。①，它们提供了一直表示数值0的方法，因为使用规格化数，我们总是必须使M >= 1。②非规格化数的另一个功能是表示那些非常接近0.0的数。它们提供了一种属性，称为逐渐溢出，其中，可能的数组均匀地接近于0.0。

  3. 特殊值

    当阶码全为1时，表示特殊值。①当小数域全为0时，表示无穷，当s = 0，表示+∞，或当s = 1时是 -∞。当我们把两个非常大的数相乘，或者除以0时，无穷能表示溢出的结果。②当小数域非零时，表示NaN(Not a Number)。运算的结果不是实数或无穷，就会返回NaN，比如：'字符' / 2，∞ - ∞。

数字示例

以6位浮点格式为例，我们能发现这个规律：①非格式化数聚集在0的附近②越靠近原点它们越密集

以8位浮点格式为例，我们要认真体会 ①机器码转换为浮点数值的过程②为什么非规格化值要把阶码值定义为E = 1 - Bais（非规格化值平滑过渡到规格化值）

我们可以通过具体示例来看看非规格化数与规格化数之间如何平滑地过渡，根据前面的原理我们还能得出一个结论，就是非规格数始终会比规格化数小。那么他们之间的过渡便可以看成是从最大非规格化数过渡到最小规格化数了（假设数值都是大于0的）。在数值大于0的情况下，最大非规格化数表示为 0 0000 111，最小规格化数表示为0 0001 000。 由于规格化数的二进制从 0 0000 111过渡到0 0001 000，我们看到0 0001 000 的阶码位 0001刚好是E = e - Bias = 1 - 7 = -6 ，而我们的最大非规格化数如何才能在指数上达成一致呢？答案是直接定义E = 1 - Bias = 1 - 7 = -6。而由于最大非规格化的已经是111，也就是说它是3位二进制的最大值7，不过我们要知道规格化数默认有数值1，相当于1000表示二进制的8。通过计算我们可以知道这就实现了最大非格式化数7 / 512 和最小规格化数8 / 512的平滑过渡。

练一练

1.已知float型变量x的机器数为BEE00000H，求x的值是多少？
答：首先我们把16进制转换为2进制，并分成三段
1　01111101　11000000000000000000000
S　E　　　　　M
计算方式： (-1)s * (1 + 尾数) * 2(阶码的值 - 127)
符号：1(负数)
阶(指数): ①阶码：01111101B = 125 ②阶码的值：125 - Bais(127) = -2
尾数: 1 + 1 * 2-1 + 1 * 2-2 + 0 * 2-3 + … = 1 + 2-1 + 2-2 = 1 + 0.5 + 0.25 = 1.75
结果：-1.75 * 2-2 = -0.4375

2.已知float型变量x的值为-12.75，求x的机器数是多少?

-12.75 = -1100.11B = 1.10011B * 23

符号S = 1
阶码E = 127 + 3 = 128 +2 = 1000 0010
显式表示的部分尾数 = 把10011抄下来，不够23位补0 = 100 1100 0000 0000 0000 0000
x的机器数表示为： 1　1000 0010　100 1100 0000 0000 0000 0000

解答为什么0.1 + 0.2 != 0.3

前面说了一堆如何用二进制表示浮点数，那么我们来考虑如何表示0.1。很可惜0.1无法精确的表示成这种形式，我们只能用另一个数尽可能接近0.1。(我们无法在10进制表示1/3，但可以在3进制精确表示它 1 * 3 -1也是这个理由)。
我们可以用一段C语言代码来研究十进制的0.1倍存储成什么：

#include<stdio.h>
void printBits(size_t const size, void const * const ptr) &#123;
    unsigned char *b = (unsigned char*) ptr;
    unsigned char byte;
    int i, j;
    for (i = size - 1; i >= 0; i --) &#123;
        for (j = 7; j >= 0; j --)&#123;
            byte = (b[i] >> j) & 1;
            printf("%u",byte);
        &#125;
    &#125;
    puts("");
&#125;

int main () &#123;
    double a = 0.1;
    double b = 0.2;
    printBits(sizeof(a),&a);
    printBits(sizeof(b),&b);
&#125;
